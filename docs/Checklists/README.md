# Review checklists (LaTeX)

**Course-style checklists** used to self-review or peer-review documentation and code before submission. Each file is a standalone LaTeX checklist aligned with a document type or milestone.

| Checklist | Typical use |
|-----------|-------------|
| `GettingStarted-Checklist.tex` | Onboarding / first steps. |
| `ProbState-Checklist.tex` | Problem statement completeness. |
| `POC-Checklist.tex` | Proof-of-concept stage. |
| `DevPlan-Checklist.tex` | Development plan. |
| `SRS-Checklist.tex` | General SRS review. |
| `SRS-SciComp-Checklist.tex` | Scientific-computing SRS variant. |
| `MG-Checklist.tex` | Module Guide. |
| `MIS-Checklist.tex` | Module Interface Specification. |
| `VnV-Checklist.tex` | Verification and validation artifacts. |
| `Code-Checklist.tex` | Code quality and standards. |
| `Writing-Checklist.tex` | Writing clarity and consistency. |
| `FinalDoc-Checklist.tex` | Final documentation package. |

Compile with your LaTeX environment as needed; these are templates, not built by the top-level `docs/Makefile` unless you add targets.
