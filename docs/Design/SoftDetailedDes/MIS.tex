\section{Introduction}

This document presents the Module Interface Specifications (MIS) for VoiceBridge, a modular platform 
designed to facilitate real-time voice interaction and transcription across diverse applications. 
VoiceBridge integrates advanced speech-to-text processing, natural language understanding, 
and command execution to enable intuitive voice-driven workflows.

The system supports accessibility standards, personalized user settings, and feedback mechanisms, 
allowing both developers and end-users to interact with software efficiently and securely. 
Core capabilities include continuous audio streaming, intent interpretation, contextual command 
mapping, and encrypted storage of transcripts and user profiles.

The MIS provides detailed specifications for each module, outlining their interfaces, expected inputs 
and outputs, state management, and interactions with other components. This document complements 
the System Requirements Specification (SRS) and Module Guide (MG), offering a reference for 
implementation, testing, and integration.

Complementary documents include the System Requirement Specifications
and Module Guide.  The full documentation and implementation can be
found at {https://github.com/speech-buddies/VoiceBridge/blob/main/docs}



\section{Notation}

The structure of the MIS for modules follows \citet{HoffmanAndStrooper1995},
with template adaptations from \citet{GhezziEtAl2003}.  
The mathematical notation used throughout this document is consistent with the
formal conventions presented in Chapter 3 of \citet{HoffmanAndStrooper1995}.
For example, the symbol := denotes multiple assignment, and conditional rules
appear in the form $(c_1 \Rightarrow r_1 \mid c_2 \Rightarrow r_2 \mid \dots \mid c_n \Rightarrow r_n)$.

This section summarizes the primitive and derived data types used by
the VoiceBridge system.

\begin{center}
\renewcommand{\arraystretch}{1.2}
\noindent
\begin{tabular}{l l p{7.5cm}}
\toprule
\textbf{Data Type} & \textbf{Notation} & \textbf{Description} \\
\midrule
character & char & A single UTF--8 encoded character. Used for transcript tokens, UI labels, and encoded metadata fields.\\
integer & $\mathbb{Z}$ & Integer values used for counters, timestamps, retry counts, audio frame lengths, and configuration parameters. \\
natural number & $\mathbb{N}$ & Positive integer values used for unique identifiers, session IDs, buffer sizes, and timeout values. \\
real & $\mathbb{R}$ & Numerical values used for confidence scores, normalized audio energy, contrast ratios, and timing measurements (seconds).\\
boolean & bool & Logical value in \{true, false\}. Used frequently across validation, VAD detections, policy checks, etc.\\
\bottomrule
\end{tabular}
\end{center}

\noindent
In addition to these primitive types, VoiceBridge uses several derived types relevant
to speech processing, browser automation, and interaction workflows:

\begin{itemize}
    \item \textbf{Sequence} — an ordered list of elements of the same type.  
    Used for sequences of audio frames, transcripts, UI messages, or system logs.

    \item \textbf{String} — a sequence of characters (UTF--8).  
    Used for transcripts, command text, error messages, ARIA labels, and browser actions.

    \item \textbf{Tuple} — fixed-length heterogeneous group of values.  
    Used for configuration records, model parameters, recognized intents, and command mappings.

    \item \textbf{Map / Dictionary} — key–value associations.  
    Commonly used for storing UI elements, active sessions, VAD states, feedback items, command registries, and structured metadata.

    \item \textbf{UUID} — universally unique identifier.  
    Used for session IDs, command IDs, feedback items, error events, and audit log entries.

    \item \textbf{AudioFrame} — a fixed-duration slice of PCM audio sampled at the engine’s operating rate (typically 16 kHz).  
    Used by MicrophoneManager, VADNoiseFilter, and SpeechToTextEngine.

    \item \textbf{Transcript} — structured object containing recognized text and confidence metadata.  
    Produced by the Speech-to-Text Engine and consumed by the Intent Interpreter.

    \item \textbf{Intent} — structured semantic representation of a user command.  
    Contains interpreted intent name, slots, and confidence score.

    \item \textbf{Command} — validated, executable instruction produced by CommandMapping and consumed by the Execution Layer.

    \item \textbf{BackendReq / BackendResp} — typed request/response objects exchanged with the Browser Orchestrator and automation bridge.

    \item \textbf{ValidationReport} — structured accessibility validation result (for example: pass/fail flag, list of issues, and recommended fixes).
\end{itemize}

\noindent
Functions used by VoiceBridge are typed by input and output domains.  
Local functions are documented in each module by listing their type signatures followed by their specification.
Where relevant, imported functions (e.g., VAD filters, ASR decoders, encryption routines, browser automation calls) are abstracted using their formal types rather than implementation details.

\section{(M1) MIS of User Interface Module} \label{M1}


\textbf{Module:} UserInterface

\subsection{Uses:}
\begin{itemize}
  \item Accessibility Layer (\hyperref[M2]{M2}) to ensure UI semantics, ARIA roles, and announcements.
  \item Feedback Display Module (\hyperref[M3]{M3}) to present messages, prompts, and recovery options.
  \item Command Orchestrator (\hyperref[M5]{M5}) for forwarding validated user events.
  \item Browser Orchestrator (\hyperref[M7]{M7}) / rendering engine to perform DOM updates and capture input events.
\end{itemize}

\subsection{Syntax:}

\subsubsection{Exported Constants:}
\begin{itemize}
  \item None.
\end{itemize}

\subsubsection{Exported Access Programs:}

\begin{tabular}{|l|l|l|l|}
\hline
\textbf{Name} & \textbf{In} & \textbf{Out} & \textbf{Exceptions} \\ \hline
\texttt{UserInterface} & \texttt{config: UiConfig} & \texttt{self: UserInterface} & \texttt{InitializationError} \\ 
\texttt{receiveInput} & \texttt{event: UiEvent} & - & \texttt{InputError} \\ 
\texttt{render} & \texttt{state: UiState} & - & \texttt{RenderError} \\ 
\texttt{showFeedback} & \texttt{msg: FeedbackItem} & - & - \\ 
\texttt{setFocus} & \texttt{elem\_id: string} & - & - \\ \hline
\end{tabular}

\subsection{Semantics}

\subsubsection{State Variables:}
\begin{itemize}
  \item \texttt{currentState}: \texttt{UiState} — current layout, visible components, and active feedback.
  \item \texttt{config}: \texttt{UiConfig} — persisted UI preferences (theme, verbosity).
  \item \texttt{focus\_target}: \texttt{string | null} — element currently targeted for keyboard/screen-reader focus.
\end{itemize}

\subsubsection{Environment Variables:}
\begin{itemize}
  \item \texttt{UI\_THEME} — runtime theme selection (light/dark).
  \item \texttt{LANG} — active locale.
\end{itemize}

\subsubsection{Assumptions:}
\begin{itemize}
  \item Browser rendering engine and event APIs are available and conform to expected semantics.
  \item Downstream modules (\hyperref[M4]{M4}--\hyperref[M6]{M6}) accept events in the documented formats.
\end{itemize}
\subsubsection{Access Routine Semantics}

\noindent\texttt{UserInterface(config)}:
\begin{itemize}
\item \textbf{transition:} Initialize \texttt{currentState} and \texttt{config}; bind to Accessibility Layer (\hyperref[M2]{M2}) and Feedback Display (\hyperref[M3]{M3}).
\item \textbf{output:} Initialized UI instance.
\item \textbf{exception:} \texttt{InitializationError} if required resources are unavailable.
\end{itemize}

\noindent\texttt{receiveInput(event)}:
\begin{itemize}
\item \textbf{transition:} Validate \texttt{event}; update \texttt{currentState} or forward to Command Orchestrator (\hyperref[M5]{M5}) where appropriate.
\item \textbf{output:} -
\item \textbf{exception:} \texttt{InputError} if \texttt{event} is malformed or unsupported.
\end{itemize}

\noindent\texttt{render(state)}:
\begin{itemize}
\item \textbf{transition:} Reconcile \texttt{currentState} with \texttt{state}; update DOM/renderer; notify Accessibility Layer (\hyperref[M2]{M2}) of attribute changes.
\item \textbf{output:} -
\item \textbf{exception:} \texttt{RenderError} on failure.
\end{itemize}

\noindent\texttt{showFeedback(msg)}:
\begin{itemize}
  \item \textbf{transition:} Delegate presentation to Feedback Display Module (\hyperref[M3]{M3}); ensure accessible announcement via Accessibility Layer (\hyperref[M2]{M2}).
  \item \textbf{output:} -
  \item \textbf{exception:} -
\end{itemize}

\noindent\texttt{setFocus(elem\_id)}:
\begin{itemize}
  \item \textbf{transition:} Set keyboard and screen-reader focus to element identified by \texttt{elem\_id}; update \texttt{focus\_target} state.
  \item \textbf{output:} -
  \item \textbf{exception:} \texttt{InputError} if \texttt{elem\_id} does not exist.
\end{itemize}

\subsubsection{Local Functions}

\begin{itemize}
  \item \texttt{apply\_config}(\texttt{UiConfig cfg}): \texttt{UiConfig} $\rightarrow$ \texttt{UiState} \\
  \textbf{Description:} \texttt{apply\_config(cfg)} returns the UI state after setting theme, verbosity, and other options to match \texttt{cfg}.

  \item \texttt{normalize\_event}(\texttt{RawUiEvent e}): \texttt{RawUiEvent} $\rightarrow$ \texttt{UiEvent} \\
  \textbf{Description:} \texttt{normalize\_event(e)} returns a standard \texttt{UiEvent} with type, target, and payload from raw event \texttt{e}.
\end{itemize}

\section{(M2) MIS of Accessibility Layer} \label{M2}
\textbf{Module:} AccessibilityLayer

\subsection{Uses:}
\begin{itemize}
  \item User Interface (\hyperref[M1]{M1}) to read and modify UI elements and attributes.
  \item WCAG guidance / accessibility utilities to verify contrast, labeling, and keyboard support.
  \item Feedback Display (\hyperref[M3]{M3}) to coordinate announcements for user messages.
\end{itemize}

\subsection{Syntax:}

\subsubsection{Exported Constants:}
\begin{itemize}
  \item None.
\end{itemize}

\subsubsection{Exported Access Programs:}

\begin{tabular}{|l|l|l|l|}
\hline
\textbf{Name} & \textbf{In} & \textbf{Out} & \textbf{Exceptions} \\ \hline
\texttt{AccessibilityLayer} & \texttt{parent: UserInterface} & self & - \\ 
\texttt{applySettings} & \texttt{settings: AccessConfig} & bool & - \\ 
\texttt{announce} & \texttt{msg: string} & \texttt{status: string} & - \\ 
\texttt{validateElement} & \texttt{elem\_id: string} & \texttt{ValidationReport} & - \\ \hline
\end{tabular}

\subsection{Semantics}

\subsubsection{State Variables:}
\begin{itemize}
  \item \texttt{settings: AccessConfig} — active accessibility options (font scale, contrast overrides, ARIA mappings).
  \item \texttt{live\_region\_id: string} — identifier for announcement region.
\end{itemize}

\subsubsection{Environment Variables:}
\begin{itemize}
  \item \texttt{WCAG\_LEVEL} — target conformance level (e.g., AA).
\end{itemize}

\subsubsection{Assumptions:}
\begin{itemize}
  \item Parent UI (\hyperref[M1]{M1}) exposes element identifiers and supports attribute updates.
  \item Localization resources exist for accessible labels when required.
\end{itemize}
\subsubsection{Access Routine Semantics}

\noindent\texttt{applySettings(settings)}:
\begin{itemize}
\item \textbf{transition:} Merge provided \texttt{settings} with defaults; apply text scaling, contrast adjustments, and ARIA attribute mappings via \texttt{parent}.
\item \textbf{output:} Boolean success status indicating whether settings were applied successfully.
\item \textbf{exception:} -.
\end{itemize}

\noindent\texttt{announce(msg)}:
\begin{itemize}
\item \textbf{transition:} Post \texttt{msg} to the live region and/or invoke screen-reader API for immediate announcement.
\item \textbf{output:} Confirmation token or status indicating the announcement was successfully scheduled or posted.
\item \textbf{exception:} -.
\end{itemize}

\noindent\texttt{validateElement(elem\_id)}:
\begin{itemize}
  \item \textbf{transition:} Inspect UI element attributes (role, label, states); compute a \texttt{ValidationReport} capturing compliance with accessibility standards.
  \item \textbf{output:} \texttt{ValidationReport} object detailing any missing roles, labels, or contrast issues.
  \item \textbf{exception:} -.
\end{itemize}

\subsubsection{Local Functions}

\begin{itemize}
  \item \texttt{check\_contrast}(\texttt{rgb fg, rgb bg}): \texttt{rgb} $\times$ \texttt{rgb} $\rightarrow$ \texttt{bool} \\
  \textbf{Description:} \texttt{check\_contrast(fg, bg)} $\equiv$ \(\frac{L_1 + 0.05}{L_2 + 0.05}\) $\geq$ \texttt{WCAG\_THRESHOLD} where $L_1, L_2$ are relative luminances of fg/bg (used by \texttt{validateElement})

  \item \texttt{aria\_set}(\texttt{string elemId, AccessMetadata metadata}): \texttt{string} $\times$ \texttt{AccessMetadata} $\rightarrow$ \texttt{bool} \\
  \textbf{Description:} \texttt{aria\_set(elemId, metadata)} returns \texttt{true} if ARIA attributes from \texttt{metadata} are applied to element \texttt{elemId} (used by \texttt{applySettings})
\end{itemize}

\noindent\texttt{clear()}:
\begin{itemize}
  \item \textbf{transition:} remove all entries from \texttt{messages} and update UI.
  \item \textbf{output:} -.
  \item \textbf{exception:} -.

\noindent\texttt{makeRecovery(feedbackId)}:
\begin{itemize}
  \item \textbf{transition:} build interactive recovery options (buttons, suggested actions) for the feedback item.
  \item \textbf{output:} \texttt{RecoveryOptions}.
  \item \textbf{exception:} -.
\end{itemize}

  \textbf{Description:} \texttt{format\_message(msg, type)} returns a \texttt{FeedbackItem} with style, icon, and metadata matching \texttt{type} for text \texttt{msg}.

  \item \texttt{schedule\_dismiss}(\texttt{UUID feedbackId, N ttl\_s}): \texttt{UUID} $\times$ \texttt{N} $\rightarrow$ \texttt{bool} \\
  \textbf{Description:} \texttt{schedule\_dismiss(feedbackId, ttl\_s)} returns \texttt{true} if timer is set to remove feedback \texttt{feedbackId} after \texttt{ttl\_s} seconds.
\end{itemize}

\section{(M4) MIS of Speech-to-Text Engine}
\label{M4}

\textbf{Module:} SpeechToTextEngine

\subsection{Uses:}
\begin{itemize}
  \item Audio capture interface to receive microphone streams.
  \item Noise filtering and Voice Activity Detection (VAD) for preprocessing.
  \item Error Feedback (\hyperref[M6]{M6}) for reporting processing failures.
  \item AuditLogger (\hyperref[M11]{M11}) for logging processing events and errors.
  \item Command Orchestrator (\hyperref[M5]{M5}) for integration with downstream modules.
  \item Loading ASR model for speech-to-text processing.
\end{itemize}

\subsection{Syntax:}

\subsubsection{Exported Constants:}
\begin{itemize}
  \item \texttt{EXPECTED\_SAMPLE\_RATE: int = 16000} — Expected audio sample rate in Hz.
  \item \texttt{EXPECTED\_FORMAT: str = "PCM\_16K\_MONO"} — Expected audio format.
\end{itemize}


\subsubsection{Exported Access Programs}

\begin{center}
\begin{tabularx}{\textwidth}{|l|>{\raggedright\arraybackslash\ttfamily}X|l|l|}
\hline
\textbf{Name} & \textbf{In} & \textbf{Out} & \textbf{Exceptions} \\ \hline

SpeechToTextEngine &
config: AsrConfig, model\_endpoint: str, api\_key: str, device: str, vad: VadInterface, noise\_filter: NoiseFilterInterface
& self & InitializationError \\

processAudio & audioData: AudioStream & Transcript & ProcessingError \\
reset & - & - & - \\
validateAudioFormat & audioData: AudioStream & bool & - \\
validateModelReady & - & bool & ModelValidationError \\ \hline
\end{tabularx}
\end{center}


\subsection{Semantics}

\subsubsection{State Variables:}
\begin{itemize}
  \item \texttt{config: AsrConfig} — engine parameters (sample rate, frame size, thresholds).
  \item \texttt{model: AsrModel} — loaded acoustic/language models and decoder state.
  \item \texttt{audio\_buffer: AudioBuffer} — buffered input audio awaiting processing.
\end{itemize}

\subsubsection{Environment Variables:}
\begin{itemize}
  \item \texttt{ASR\_CLOUD\_ENDPOINT} — URL of the cloud-hosted ASR model.
  \item \texttt{ASR\_API\_KEY} — API key for authenticating with the cloud-hosted ASR model.
\end{itemize}

\subsubsection{Assumptions:}
\begin{itemize}
  \item Input audio meets expected format and sample rate.
  \item The ASR model is accessible via the provided endpoint and API key.
\end{itemize}

\subsubsection{Access Routine Semantics:}

\noindent\texttt{SpeechToTextEngine(config, model\_endpoint, api\_key, device, vad, noise\_filter)}:
\begin{itemize}
  \item \textbf{transition:} Initialize engine internals, allocate \texttt{audio\_buffer}, and set \texttt{model\_endpoint} and \texttt{api\_key}.
  \item \textbf{output:} Initialized engine instance.
  \item \textbf{exception:} \texttt{InitializationError} if the endpoint or API key is invalid or inaccessible.
\end{itemize}

\noindent\texttt{processAudio(audioData)}:
\begin{itemize}
  \item \textbf{transition:} Validate audio format, preprocess (VAD, noise suppression), send audio to the cloud-based ASR model via \texttt{model\_endpoint} with \texttt{api\_key}, and decode the response.
  \item \textbf{output:} \texttt{Transcript} containing recognized text and confidence metadata.
  \item \textbf{exception:} \texttt{ProcessingError} if format invalid, preprocessing fails, or the cloud model is unreachable.
\end{itemize}

\noindent\texttt{reset()}:
\begin{itemize}
  \item \textbf{transition:} Clear \texttt{audio\_buffer}.
  \item \textbf{output:} -.
  \item \textbf{exception:} -.
\end{itemize}

\noindent\texttt{validateAudioFormat(audio)}:
\begin{itemize}
  \item \textbf{transition:} None.
  \item \textbf{output:} \texttt{true} iff \texttt{audio.sampleRate = EXPECTED\_SAMPLE\_RATE $\wedge$ audio.format = EXPECTED\_FORMAT}.
  \item \textbf{exception:} -.
\end{itemize}

\noindent\texttt{validateModelReady()}:
\begin{itemize}
  \item \textbf{transition:} None.
  \item \textbf{output:} \texttt{true} iff the cloud-based ASR model is accessible via \texttt{model\_endpoint} and the \texttt{api\_key} is valid.
  \item \textbf{exception:} \texttt{ModelValidationError} if the endpoint is unreachable or the API key is invalid.
\end{itemize}

\subsubsection{Local Functions:}
\begin{itemize}
  \item \texttt{extract\_features(audio: AudioStream) -> FeatureVector}: Returns feature vector (e.g., spectrogram) computed from input audio.
  \item \texttt{send\_to\_cloud(features: FeatureVector) -> Transcript}: Sends the feature vector to the cloud-based ASR model using the \texttt{model\_endpoint} and \texttt{api\_key}, and returns the decoded transcript.
\end{itemize}

\textbf{Module:} CommandOrchestrator

\subsection{Uses:}
\begin{itemize}

 \item User Profile Manager (\hyperref[M8]{M8}) for personalized context and preferences.
  \item Error Feedback Module (\hyperref[M6]{M6}) for reporting API or schema validation failures.
  \item Security Layer (\hyperref[M9]{M9}) for managing API credentials and encryption.
\end{itemize}

\subsection{Syntax:}

\subsubsection{Exported Constants:}
\begin{itemize}
  \item None — Configuration values are passed via constructor parameters or loaded from system prompt.
\end{itemize}

\subsubsection{Exported Access Programs:}

\begin{tabular}{|l|l|l|l|}
\hline
\textbf{Name} & \textbf{In} & \textbf{Out} & \textbf{Exceptions} \\ \hline
\texttt{CommandOrchestrator} & - & self & InitializationError \\ 
\texttt{process} & \begin{tabular}[t]{@{}l@{}}\texttt{user\_input: str} \\ \texttt{conversation\_context:} \\ \texttt{\hspace{1em}Optional[List[dict]]}\end{tabular} & \texttt{OrchestratorResponse} & \texttt{InferenceError} \\
\texttt{reset} & - & - & - \\ 
\texttt{apply\_guardrails} & \texttt{command: String} & \texttt{(bool, String)} & - \\ \hline
\end{tabular}

\subsection{Semantics}

\subsubsection{State Variables:}
\begin{itemize}
  \item apiKey: stores API key for LLM service authentication
  \item client: LLM API client instance for command clarification
  \item modelId: identifier for selected LLM model
  \item systemPrompt: current system prompt instructions for LLM
  \item conversationHistory: list of user/assistant turns for multi-turn context
  \item currentGoal: most recent clarified user intent (if any)
\end{itemize}

\subsubsection{Environment Variables:}
\begin{itemize}
  \item \texttt{GEMINI\_API\_KEY} — API key for Google Gemini service (used if api\_key parameter not provided to constructor)
\end{itemize}

\subsubsection{Assumptions:}
\begin{itemize}
  \item LLM returns valid JSON in format \texttt{\{"needs\_clarification": bool, "question": String\}} or \texttt{\{"needs\_clarification": bool, "clarified\_command": String\}}
  \item System prompt effectively guides the LLM to ask for clarification only when truly necessary
  \item Browser controller can handle natural language commands
\end{itemize}

\subsubsection{Access Routine Semantics}

\noindent\texttt{CommandOrchestrator(api\_key, model\_id, prompt\_path)}:
\begin{itemize}
  \item \textbf{transition:} Load environment variables; initialize client, set \texttt{model\_id} (default "gemini-2.5-flash"); load \texttt{system\_prompt} from \texttt{prompt\_path} if provided, otherwise use default prompt; initialize empty \texttt{conversation\_history}; set \texttt{current\_goal} to None.
  \item \textbf{output:} Initialized orchestrator instance.
  \item \textbf{exception:} \texttt{InitializationError} if API key is missing or config is malformed.
\end{itemize}

\noindent\texttt{process(user\_input, conversation\_context)}:
\begin{itemize}
\item \textbf{transition:} Build message list: add messages from \texttt{conversation\_context} if provided, then append \texttt{user\_input} as new user message; call \texttt{\_call\_llm(messages)} to send to Gemini with \texttt{system\_prompt}; parse JSON response using \texttt{\_parse\_response()}; construct and return OrchestratorResponse.
\item \textbf{output:} \texttt{OrchestratorResponse} object with fields: \texttt{needs\_clarification} (bool), \texttt{user\_prompt} (optional question for user), \texttt{clarified\_command} (optional natural language command), \texttt{reasoning} (optional debug info), \texttt{metadata} (optional dict with "original\_input" and other context).
\item \textbf{exception:} \texttt{InferenceError} if LLM API call fails, times out, or returns unparsable response.
\end{itemize}

\noindent\texttt{reset()}:
\begin{itemize}
\item \textbf{transition:} Clear \texttt{conversation\_history} to empty list; set \texttt{current\_goal} to None.
\item \textbf{output:} None.
\item \textbf{exception:} None.
\end{itemize}

\noindent\texttt{apply\_guardrails(command)} [static method]:
\begin{itemize}
\item \textbf{transition:} None (pure function).
\item \textbf{output:} Tuple \texttt{(allowed: bool, message: String)} where \texttt{allowed} is \texttt{False} if command contains blocked keywords AND is phrased as an imperative action; \texttt{message} contains "Sorry, this command cannot be executed because it violates safety policy." if blocked, None otherwise.
\item \textbf{exception:} None.
\end{itemize}

\subsubsection{Local Functions:}
\begin{itemize}
  \item \texttt{\_get\_default\_system\_prompt()}: \texttt{void} $\rightarrow$ \texttt{String} \\
  \textbf{Description:} Returns default system prompt containing instructions for command clarification, JSON output format specification, examples of when to ask clarification vs. when to proceed, and guidance to ignore filler words while extracting user intent.

  \item \texttt{\_call\_llm(messages: List[Dict[String, String]])}: \texttt{List[Dict]} $\rightarrow$ \texttt{String} \\
  \textbf{Description:} Constructs full message list with \texttt{system\_prompt} as first user message, then appends all messages from conversation history (converting "user"/"assistant" roles to "user"/"model" for Gemini API); returns response text.

  \item \texttt{\_parse\_response(response\_text: String, original\_input: String)}: \texttt{String} $\times$ \texttt{String} $\rightarrow$ \texttt{OrchestratorResponse} \\
  \textbf{Description:} Strips markdown code fences from response\_text; parses JSON; if \texttt{needs\_clarification=true}, returns OrchestratorResponse with \texttt{user\_prompt} set to question from JSON; if \texttt{needs\_clarification=false}, returns OrchestratorResponse with \texttt{clarified\_command} from JSON; if parsing fails, returns fallback OrchestratorResponse requesting clarification with error details in \texttt{reasoning} and \texttt{metadata}.
\end{itemize}

\subsection{Semantics}

Implements audio acquisition requirements (FR1). Provides a self-contained, callback-driven pipeline from raw microphone input to segmented, normalized utterances ready for ASR processing.

\subsubsection{State Variables:}
\begin{itemize}
  \item \texttt{config: AudioConfig} — immutable capture and VAD parameters set at construction.
  \item \texttt{state: string} — current position in the \texttt{AudioState} machine; protected by \texttt{state\_lock}.
  \item \texttt{pre\_buffer: deque[bytes]} — ring buffer of the most recent \texttt{pre\_buffer\_chunks} PCM chunks, used to seed recordings with audio that preceded speech detection.
  \item \texttt{recording\_buffer: list[bytes]} — accumulates raw PCM bytes for the utterance currently being recorded.
  \item \texttt{silence\_counter: $\mathbb{N}$} — number of consecutive silent chunks observed during RECORDING; reset on any speech frame.
  \item \texttt{recording\_chunks\_count: $\mathbb{N}$} — total chunks added to \texttt{recording\_buffer} for the current utterance.
  \item \texttt{stats: dict} — running counters: \texttt{total\_recordings}, \texttt{total\_audio\_seconds}, \texttt{vad\_errors}, \texttt{stream\_errors}.
\end{itemize}

\subsubsection{Environment Variables:}
\begin{itemize}
  \item System audio input device (selected via \texttt{config.device} or OS default).
\end{itemize}

\subsubsection{Assumptions:}
\begin{itemize}
  \item The host OS grants microphone permission to the process.
  \item The \texttt{sounddevice} library can open a stream at the configured sample rate and block size.
  \item The caller's \texttt{on\_audio\_ready} callback does not block the audio thread for extended periods.
\end{itemize}

\subsubsection{Access Routine Semantics:}

\noindent\texttt{AudioConfig(\ldots)}:
\begin{itemize}
  \item \textbf{transition:} Store all parameters; derive \texttt{chunk\_size}, \texttt{pre\_buffer\_chunks}, \texttt{silence\_chunks}, \texttt{max\_recording\_chunks} from duration inputs.
  \item \textbf{output:} Initialized \texttt{AudioConfig} record.
  \item \textbf{exception:} \texttt{ValueError} (raised by \texttt{RealtimeAudioCapture}) if \texttt{validate()} returns \texttt{false}.
\end{itemize}

\noindent\texttt{RealtimeAudioCapture(config, on\_audio\_ready)}:
\begin{itemize}
  \item \textbf{transition:} Validate \texttt{config}; initialize VAD, state machine, pre-roll buffer, recording buffer, counters, and stats.
  \item \textbf{output:} Initialized capture instance with \texttt{state = IDLE}.
  \item \textbf{exception:} \texttt{ValueError} if \texttt{config.validate()} returns \texttt{false}.
\end{itemize}

\noindent\texttt{start()}:
\begin{itemize}
  \item \textbf{transition:} Set \texttt{state := LISTENING}; open a \texttt{sounddevice.InputStream} with \texttt{\_audio\_callback} and start it. No-op if \texttt{state $\neq$ IDLE}.
  \item \textbf{output:} --
  \item \textbf{exception:} Propagates \texttt{Exception} from \texttt{sounddevice} if the device cannot be opened; reverts \texttt{state} to IDLE.
\end{itemize}

\noindent\texttt{stop()}:
\begin{itemize}
  \item \textbf{transition:} Set \texttt{state := STOPPED}; signal \texttt{stop\_event}; stop and close the stream; set \texttt{state := IDLE}.
  \item \textbf{output:} --
  \item \textbf{exception:} --
\end{itemize}

\noindent\texttt{get\_stats()}:
\begin{itemize}
  \item \textbf{transition:} None (read-only snapshot).
  \item \textbf{output:} Shallow copy of \texttt{stats} dict with keys \texttt{total\_recordings: $\mathbb{N}$}, \texttt{total\_audio\_seconds: $\mathbb{R}$}, \texttt{vad\_errors: $\mathbb{N}$}, \texttt{stream\_errors: $\mathbb{N}$}.
  \item \textbf{exception:} --
\end{itemize}

\noindent\texttt{list\_audio\_devices()} [static]:
\begin{itemize}
  \item \textbf{transition:} None.
  \item \textbf{output:} List of \texttt{(index: $\mathbb{N}$, device\_info: dict)} tuples for all input-capable devices; also prints a formatted table to stdout.
  \item \textbf{exception:} --
\end{itemize}

\subsubsection{Local Functions:}
\begin{itemize}
  \item \texttt{\_set\_state(new\_state: string)}: \texttt{string} $\rightarrow$ \texttt{void} \\
  \textbf{Description:} Acquires \texttt{state\_lock}, updates \texttt{state}, and logs the transition if the state changed.

  \item \texttt{\_get\_state()}: \texttt{void} $\rightarrow$ \texttt{string} \\
  \textbf{Description:} Returns the current \texttt{state} under \texttt{state\_lock}.

  \item \texttt{\_is\_speech(audio\_chunk: bytes)}: \texttt{bytes} $\rightarrow$ \texttt{bool} \\
  \textbf{Description:} Returns \texttt{true} iff WebRTC VAD classifies the chunk as speech \emph{or} (when \texttt{energy\_threshold} $\neq$ None) the RMS energy of the chunk exceeds \texttt{energy\_threshold}.

  \item \texttt{\_process\_audio\_chunk(audio\_chunk: bytes)}: \texttt{bytes} $\rightarrow$ \texttt{void} \\
  \textbf{Description:} State machine dispatch — appends to pre-buffer or recording buffer, drives state transitions, and calls \texttt{\_end\_recording} when silence or duration limits are reached.

  \item \texttt{\_start\_recording()}: \texttt{void} $\rightarrow$ \texttt{void} \\
  \textbf{Description:} Seeds \texttt{recording\_buffer} from \texttt{pre\_buffer} and resets utterance counters.

  \item \texttt{\_end\_recording()}: \texttt{void} $\rightarrow$ \texttt{void} \\
  \textbf{Description:} Concatenates \texttt{recording\_buffer}, converts int16 PCM to float32, builds \texttt{AudioMetadata}, increments \texttt{stats}, invokes \texttt{on\_audio\_ready}, and resets utterance state.

  \item \texttt{\_audio\_callback(indata, frames, time\_info, status)}: callback $\rightarrow$ \texttt{void} \\
  \textbf{Description:} sounddevice stream callback; converts float32 block to int16 bytes and forwards to \texttt{\_process\_audio\_chunk}; increments \texttt{stream\_errors} on non-empty status.
\end{itemize}

\texttt{train}    & --                       & \texttt{List[$\mathbb{R}$]} & \texttt{RuntimeError} \\ \hline

\subsection{Semantics}

Supports Accuracy Requirement (PF-3) by enabling incremental, user-specific fine-tuning of
Whisper-small via Low-Rank Adaptation (LoRA). A seen-files ledger and experience-replay buffer
prevent catastrophic forgetting as new audio data accumulates over time.

\subsubsection{State Variables:}
\begin{itemize}
  \item \texttt{processor: WhisperProcessor | None} — Whisper feature extractor and tokenizer; set by \texttt{setup()}.
  \item \texttt{model: PeftModel | None} — Whisper-small with LoRA adapter attached; set by \texttt{setup()}.
  \item \texttt{optimizer: AdamW | None} — parameter optimizer; set by \texttt{setup()}.
  \item \texttt{train\_loader: DataLoader | None} — batched training data for the current run; set by \texttt{setup()}.
  \item \texttt{test\_loader: DataLoader | None} — batched evaluation data for the current run; set by \texttt{setup()}.
  \item \texttt{\_current\_run\_files: List[string]} — basenames of new audio files selected for this run; populated by \texttt{setup()}.
  \item \texttt{\_global\_step: $\mathbb{N}$} — cumulative batch count across all epochs in a run; used for mid-run checkpointing.
\end{itemize}

\subsubsection{Environment Variables:}
\begin{itemize}
  \item \texttt{output\_dir} filesystem path — must be writable; stores the LoRA adapter subdirectory (\texttt{\_ADAPTER\_SUBDIR}) and seen-files ledger (\texttt{\_SEEN\_FILES\_LEDGER}).
  \item \texttt{data\_dir} filesystem path — must contain the \texttt{.wav} files listed in the metadata.
  \item HuggingFace model cache — \texttt{openai/whisper-small} is downloaded on the first run.
\end{itemize}

\subsubsection{Assumptions:}
\begin{itemize}
  \item Audio files are mono or stereo PCM \texttt{.wav} at any sample rate (resampling to \texttt{sampling\_rate} is handled internally).
  \item The metadata JSONL file contains objects with at least \texttt{"wav\_file"} and \texttt{"transcript"} fields.
  \item The host machine provides sufficient RAM for CPU-based training of Whisper-small with LoRA (adapter-only parameters only; base model weights are frozen).
  \item \texttt{setup()} is called before any of \texttt{train()}, \texttt{evaluate()}, or \texttt{save()}.
\end{itemize}

\subsubsection{Access Routine Semantics:}

\noindent\texttt{WhisperLoRATrainer(\ldots)}:
\begin{itemize}
  \item \textbf{transition:} Store all configuration parameters; set \texttt{model}, \texttt{processor}, \texttt{optimizer}, \texttt{train\_loader}, \texttt{test\_loader} to \texttt{None}; initialise \texttt{\_current\_run\_files := []} and \texttt{\_global\_step := 0}; set \texttt{device := "cpu"}.
  \item \textbf{output:} Uninitialised trainer instance (requires \texttt{setup()} before use).
  \item \textbf{exception:} --
\end{itemize}

\noindent\texttt{setup()}:
\begin{itemize}
  \item \textbf{transition:} Create \texttt{output\_dir} if absent; load all metadata via \texttt{\_load\_data()}; load seen-files ledger via \texttt{\_load\_seen\_files()}; partition into \texttt{new\_data} and \texttt{old\_data}; draw replay sample via \texttt{\_sample\_replay\_data()}; record \texttt{\_current\_run\_files := basenames(new\_data)}; load \texttt{WhisperProcessor}; build/resume model via \texttt{\_build\_model()}; construct \texttt{AdamW} optimizer; if \texttt{new\_data} is non-empty, split combined data and construct \texttt{train\_loader} and \texttt{test\_loader}.
  \item \textbf{output:} \texttt{void}; side-effect: all state variables above are populated.
  \item \textbf{exception:} \texttt{FileNotFoundError} if \texttt{meta\_file} or any referenced \texttt{wav\_file} is absent (missing \texttt{wav\_file} entries are skipped with a warning rather than raising).
\end{itemize}



\noindent\texttt{train()}:
\begin{itemize}
  \item \textbf{transition:} For each epoch $e \in [1 \ldots \texttt{epochs}]$: run \texttt{\_train\_epoch(e)}, record average loss, save per-epoch checkpoint, invoke \texttt{on\_epoch\_end} if set. Increment \texttt{\_global\_step} per batch; save mid-run checkpoint every \texttt{checkpoint\_every} steps.
  \item \textbf{output:} \texttt{List[$\mathbb{R}$]} of per-epoch average training losses; empty list if \texttt{has\_new\_data = false}.
  \item \textbf{exception:} \texttt{RuntimeError} if \texttt{setup()} has not been called (\texttt{model} is \texttt{None}).
\end{itemize}

\noindent\texttt{evaluate()}:
\begin{itemize}
  \item \textbf{transition:} Set \texttt{model} to eval mode; iterate \texttt{test\_loader} with \texttt{torch.no\_grad()}; accumulate cross-entropy loss; restore train mode.
  \item \textbf{output:} Average loss $\in \mathbb{R}$ over the test set; \texttt{NaN} if no test data is available.
  \item \textbf{exception:} \texttt{RuntimeError} if \texttt{setup()} has not been called.
\end{itemize}

\noindent\texttt{save(path)}:
\begin{itemize}
  \item \textbf{transition:} Call \texttt{model.save\_pretrained(path $\vee$ \{output\_dir\}/\{\_ADAPTER\_SUBDIR\})}; on success, call \texttt{\_update\_seen\_files(\_current\_run\_files)} to append this run's filenames to the ledger.
  \item \textbf{output:} \texttt{string} — absolute path of the directory the adapter was saved to.
  \item \textbf{exception:} \texttt{RuntimeError} if \texttt{setup()} has not been called.
\end{itemize}

\noindent\texttt{has\_new\_data} (read-only property):
\begin{itemize}
  \item \textbf{transition:} None (pure read).
  \item \textbf{output:} \texttt{true} iff \texttt{len(\_current\_run\_files) > 0}.
  \item \textbf{exception:} --
\end{itemize}

\subsubsection{Local Functions:}

\begin{itemize}
  \item \texttt{\_load\_data()}: \texttt{void} $\rightarrow$ \texttt{List[(string, string)]} \\
  \textbf{Description:} Parses \texttt{meta\_file} (JSONL) and returns \texttt{(audio\_path, transcript)} pairs for all records whose \texttt{wav\_file} exists on disk; skips missing files with a warning.

  \item \texttt{\_split(data)}: \texttt{List[(string, string)]} $\rightarrow$ \texttt{(List, List)} \\
  \textbf{Description:} Shuffles \texttt{data} with \texttt{seed} and splits into train / test at the \texttt{test\_split} boundary.

  \item \texttt{\_build\_model()}: \texttt{void} $\rightarrow$ \texttt{PeftModel} \\
  \textbf{Description:} Loads \texttt{openai/whisper-small}. If \texttt{\_ADAPTER\_SUBDIR} exists inside \texttt{output\_dir}, resumes from the saved adapter (\texttt{PeftModel.from\_pretrained}); otherwise initialises a fresh \texttt{LoraConfig} targeting \texttt{q\_proj} and \texttt{v\_proj} in the decoder.

  \item \texttt{\_sample\_replay\_data(old\_data, n\_new)}: \texttt{List} $\times$ $\mathbb{N}$ $\rightarrow$ \texttt{List} \\
  \textbf{Description:} Returns a random sample of size $\min(\lfloor n\_\text{new} \times \texttt{replay\_ratio} \rfloor,\, |\texttt{old\_data}|)$ from \texttt{old\_data}; returns \texttt{[]} when \texttt{replay\_ratio} $\leq 0$ or \texttt{old\_data} is empty.

  \item \texttt{\_load\_seen\_files()}: \texttt{void} $\rightarrow$ \texttt{Set[string]} \\
  \textbf{Description:} Reads the JSON array in \texttt{\_SEEN\_FILES\_LEDGER}; returns an empty set if the ledger does not exist.

  \item \texttt{\_update\_seen\_files(new\_filenames)}: \texttt{List[string]} $\rightarrow$ \texttt{void} \\
  \textbf{Description:} Merges \texttt{new\_filenames} into the existing ledger set and writes the sorted result back to \texttt{\_SEEN\_FILES\_LEDGER}.

  \item \texttt{\_train\_epoch(epoch)}: $\mathbb{N}$ $\rightarrow$ $\mathbb{R}$ \\
  \textbf{Description:} Runs one full pass over \texttt{train\_loader}: forward pass, cross-entropy loss, \texttt{zero\_grad}, \texttt{backward}, \texttt{step}. Logs loss every \texttt{print\_every} batches, saves a checkpoint every \texttt{checkpoint\_every} global steps, fires \texttt{on\_batch\_end} after each batch. Returns average loss for the epoch.

  \item \texttt{\_require\_setup()}: \texttt{void} $\rightarrow$ \texttt{void} \\
  \textbf{Description:} Raises \texttt{RuntimeError} if \texttt{model} is \texttt{None}, guarding all public methods that depend on \texttt{setup()}.
\end{itemize}

\end{itemize}

\noindent\texttt{addRecordedCommand(clarifiedCommand)}:

\end{itemize}

\noindent\texttt{stopRecording()}:
\begin{itemize}
\item \textbf{transition:} If recording is active and \texttt{\_buffer} is non-empty, creates a new shortcut record with a unique ID, default name, and recorded command list; stores it in \texttt{\_shortcuts}; resets recording state; and flushes data to persistent storage.
\item \textbf{output:} Returns the created shortcut record.
\item \textbf{exception:} Raises \texttt{ValueError} if no recording is active or if no commands were recorded.
\end{itemize}

\noindent\texttt{cancelRecording()}:
\begin{itemize}
\item \textbf{transition:} Sets \texttt{\_recording := false} and clears \texttt{\_buffer} without saving a shortcut.
\end{itemize}

\noindent\texttt{isRecording()}:
\begin{itemize}
\item \textbf{output:} Returns the current value of \texttt{\_recording}.
\end{itemize}

\noindent\texttt{listShortcuts()}:
\begin{itemize}
\item \textbf{output:} Returns a copy of the currently stored shortcuts.
\end{itemize}

\noindent\texttt{deleteShortcut(shortcutID)}:
\begin{itemize}
\item \textbf{transition:} Removes the shortcut associated with \texttt{shortcutID} if it exists, and flushes the updated shortcut set to persistent storage.
\item \textbf{output:} Returns \texttt{true} if a shortcut was removed, otherwise \texttt{false}.
\end{itemize}
